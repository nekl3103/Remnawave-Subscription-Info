Быстрая настройка sing-box из подписки Remnawave

Что умеет программа

Все основные действия собраны в одном терминальном интерфейсе.

01

Список серверов

Загружает подписку Remnawave, показывает серверы и поддерживает поиск, фильтр и обновление списка.

02

Поддержка протоколов

Работает с VLESS, Shadowsocks, Trojan и Hysteria2. VMess поддерживается для чтения из подписок.

03

Генерация outbounds

Создает готовый outbound для sing-box и может экспортировать его в JSON-файл.

04

Запись на роутер

Подключается к OpenWrt по SSH и обновляет только outbound с тегом proxy.

05

Backup и проверка

Создает резервную копию config.json.bak и проверяет новый конфиг перед заменой.

06

Статус и логи

Показывает состояние sing-box и выводит логи сервиса в реальном времени.

Требования

Перед записью на роутер должны быть готовы базовые настройки OpenWrt и sing-box.

OW

OpenWrt

Роутер с доступом к shell и стандартным управлением сервисами.

SB

sing-box

Установленный sing-box, доступный через init.d или service.

SSH

SSH-доступ

Логин и пароль пользователя роутера, по умолчанию root@192.168.1.1.

CFG

config.json

Существующий файл /etc/sing-box/config.json с секцией outbounds.

SUB

Подписка

Рабочая ссылка Remnawave, доступная с компьютера, где запущена программа.

Как выглядит работа с сервером

Пять основных экранов CLI: от загрузки подписки до управления выбранным сервером и роутером.

Главный экран

Программа стартует в терминале, показывает версию, ссылки проекта и сразу просит вставить ссылку подписки Remnawave.

Этот экран нужен, чтобы быстро начать работу без отдельной настройки или конфигурационного файла.

Как это работает

Сценарий рассчитан на роутер OpenWrt с установленным и настроенным sing-box.

1

Вставьте ссылку подписки

Программа загрузит серверы Remnawave и подготовит список для выбора.

2

Выберите сервер

Можно посмотреть детали, отфильтровать список по протоколу или обновить подписку.

3

Подключитесь к роутеру

По умолчанию используется адрес root@192.168.1.1, но можно указать свой SSH-адрес.

4

Примените конфигурацию

Новый config.json проверяется командой sing-box check, после успеха сервис перезапускается.

FAQ и безопасность

Коротко о том, что происходит при записи конфигурации на роутер.

!

Пароль сохраняется?

SSH-пароль хранится только в памяти текущего запуска. После перезапуска программы его нужно ввести снова.

HK

Проверяется host key?

При первом SSH-подключении программа показывает fingerprint и сохраняет доверенный ключ.

?

Что меняется в config.json?

Обновляется outbound с тегом proxy. Остальные outbounds не удаляются.

✓

Есть ли backup?

Перед заменой создается резервная копия /etc/sing-box/config.json.bak, которую можно восстановить из меню.

#

Проверяется ли конфиг?

Перед сохранением используется sing-box check. Если проверка не проходит, основной конфиг не заменяется.

PR

Что именно заменяется?

Программа ищет outbound с tag: proxy и заменяет только его, не трогая остальные правила.
